# Configuration reference

Outpost reads a single YAML file, passed via `outpost serve -config <path>` (defaults to `outpost.yaml`). Every field below matches `internal/config.Config` exactly — this doc is generated by reading that struct, not reconstructed from memory.

## Top level

| Field | YAML key | Type | Required | Default | Notes |
|---|---|---|---|---|---|
| `Listen` | `listen` | string | **yes** | — | Address Outpost listens on, e.g. `"127.0.0.1:8100"`. |
| `StateDB` | `state_db` | string | no | `"outpost.db"` | Path to the SQLite state file (pins, drift history, breaker state). |
| `ControlPlaneURL` | `control_plane_url` | string | no | `""` (disabled) | Base URL of an optional hosted control plane. Empty means no reporting at all — `internal/reporter` is never constructed. |
| `ControlPlaneAPIKey` | `control_plane_api_key` | string | no | `""` | Sent as `Authorization: Bearer <key>` on every report. **Must match the control plane's own `CONTROL_PLANE_API_KEY`** if that's set — if the control plane has auth enabled and this is left empty, every report is silently 401'd and dropped (fail-silent by design; nothing will tell you this is happening except an empty `drift_events` table on the control-plane side). Ignored entirely if `control_plane_url` is unset. |
| `Upstreams` | `upstreams` | list of [Upstream](#upstream) | **yes**, at least one | — | One route per entry, at `/{name}`. |
| `Tools` | `tools` | map of tool name → [ToolOverride](#tooloverride) | no | `{}` | Per-tool opt-in overrides. Keyed by the tool's `name` as declared in its MCP definition, not the upstream name. |

## Upstream

| Field | YAML key | Type | Required | Default | Notes |
|---|---|---|---|---|---|
| `Name` | `name` | string | **yes** | — | Becomes the route path: `/{name}`. Also the key used everywhere else (breaker state, pinning, cache keys) to distinguish this upstream from others. |
| `URL` | `url` | string | **yes** | — | The real MCP server Outpost forwards to. |
| `CacheTTLSeconds` | `cache_ttl_seconds` | int | no | `0` (disabled) | TTL for the list-op cache (`tools/list`/`resources/read` only — `tools/call` is uncacheable by construction regardless of this setting). `0` or unset disables caching for this upstream. |

## ToolOverride

Keyed by tool name under the top-level `tools:` map.

| Field | YAML key | Type | Required | Default | Notes |
|---|---|---|---|---|---|
| `Retry` | `retry` | [RetryPolicy](#retrypolicy) or absent | no | `nil` (no retries) | **There is deliberately no global retry setting** — this is the only place retries can be enabled, per tool, explicitly (spec §10, enforced by a reflection test asserting `Config` has no top-level retry field). |
| `Block` | `block` | bool | no | `false` | When `true`, a `tools/call` to this tool is rejected (`DriftBlocked`, code `-32002`) once pinning has detected drift on its definition. When `false` (default), drift is logged but never blocks. |

## RetryPolicy

| Field | YAML key | Type | Required | Default |
|---|---|---|---|---|
| `MaxAttempts` | `max_attempts` | int | no | `0` |
| `InitialBackoffMS` | `initial_backoff_ms` | int | no | `0` |

## Validation

`Load` fails (and `outpost serve` refuses to start) if:
- `listen` is empty
- `upstreams` is empty
- any upstream is missing `name` or `url`

## Environment variables

Config-file fields cover per-instance MCP proxy behavior. A few cross-cutting things are environment variables instead, matching every other optional feature's "off unless explicitly set" posture:

| Variable | Used by | Effect when unset |
|---|---|---|
| (none currently — the binary reads everything from the config file) | | |

The **control plane** (`outpost-cloud`, separate repo/binary) reads its own environment variables — not part of this file, listed here for completeness since operators configure both:

| Variable | Effect when unset |
|---|---|
| `LISTEN_ADDR` | Defaults to `127.0.0.1:8200` |
| `STATE_DB` | Defaults to `controlplane.db` |
| `ANTHROPIC_API_KEY` | Scan-on-change disabled |
| `SLACK_WEBHOOK_URL` | Alert dispatch disabled |
| `CONTROL_PLANE_API_KEY` | Ingest endpoints accept any caller (not safe to expose publicly) |

## Complete example

```yaml
listen: "0.0.0.0:8100"
state_db: "/var/lib/outpost/outpost.db"
control_plane_url: "https://control-plane.internal.example.com"
control_plane_api_key: "match-the-control-plane's-CONTROL_PLANE_API_KEY-exactly"

upstreams:
  - name: files
    url: "http://127.0.0.1:9000/mcp"
    cache_ttl_seconds: 30
  - name: search
    url: "http://127.0.0.1:9001/mcp"

tools:
  files.write:
    block: true   # reject tools/call once drift is detected on this tool
  files.read:
    retry:
      max_attempts: 3
      initial_backoff_ms: 100
```
